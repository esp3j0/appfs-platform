# APPFS v0.2 总览（后端模式）

- 版本：`v0.2-draft`
- 状态：`Frozen (Phase A, 2026-03-20)`
- 语言：中文

## 1. 目标

1. 将 AppFS 从 v0.1 的 sidecar 参考模式升级为 backend-native 模式。
2. 让 AppFS 作为可对接任意 app 的统一框架，稳定支持动作执行、资源读取、事件流、分页与缓存。
3. 在保持 Agent 可文件化操作体验的前提下，提供可实现、可测试、可迁移的 v0.2 规范。

## 2. 为什么是 v0.2

本次升级是语义级变更，不是补丁：

| 方面 | v0.1 (Sidecar) | v0.2 (Backend-native) |
|------|----------------|----------------------|
| 架构模式 | 独立进程轮询 | 后端读拦截 |
| Snapshot 缓存 | 预生成或手动刷新 | 读穿式自动扩展 |
| 读路径 | 不感知读取行为 | 感知 offset/length |
| 缓存 miss | 返回空或旧数据 | 自动触发上游拉取 |
| 并发控制 | 无保证 | 资源级去重锁 |
| 恢复能力 | 无 | Journal 持久化 |
| 适用场景 | Demo/验证 | 生产环境 |

## 3. v0.1 与 v0.2 边界

### 3.1 v0.1（保留）

1. **定位**：demo/reference。
2. **价值**：协议验证、开发者体验验证、回归对照基线。
3. **约束**：不再承接架构级新能力。

### 3.2 v0.2（新增主路线）

1. **定位**：可生产演进的 backend 模式。
2. **能力**：读路径拦截、快照预热与扩容、持久化状态恢复、并发去重。
3. **策略**：先文档冻结，再进入实现。

## 4. 框架范围（任意 app 可接入）

v0.2 框架职责：

| 职责 | 说明 |
|------|------|
| 统一动作提交协议 | `.act`，JSONL 行协议 |
| 统一资源语义 | snapshot：`*.res.jsonl` 全文件；live：`*.res.json` 分页 |
| 统一事件与重放语义 | events/replay/cursor |
| 统一错误码 | 最小错误码集 + 扩展错误码 |

不在 v0.2 核心内强制：

| 不强制项 | 说明 |
|----------|------|
| 任意上游 API 风格 | REST/gRPC/SDK 都可 |
| 任意 app 领域模型 | 聊天、CRM、任务系统均可 |
| Connector 传输形态 | in-process/HTTP/gRPC 均可 |

## 5. 关键术语

| 术语 | 定义 |
|------|------|
| **snapshot** | 有限数据资源，面向全文检索，暴露为 `*.res.jsonl` |
| **live** | 动态数据资源，暴露为 `*.res.json` 分页视图 |
| **预热 (prewarm)** | 启动阶段提前拉取 snapshot 元信息（如 size/revision） |
| **读穿扩展 (read-through)** | 读取超出缓存时触发上游拉取并扩展缓存 |
| **物化 (materialization)** | 将上游数据转为本地文件语义（JSONL/JSON）的过程 |
| **游标 (cursor)** | 顺序消费与重放定位信息 |
| **Journal** | 请求/状态持久化记录，用于恢复与审计 |
| **Connector** | 对接具体 app 的适配层（每个 app 一个实现） |

## 7. 核心配置项

### 7.1 预热配置

```json
"snapshot": {
  "prewarm": true,
  "prewarm_timeout_ms": 5000
}
```

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `prewarm` | `true` | 是否在启动时预热 |
| `prewarm_timeout_ms` | `5000` | 单资源预热超时 |

### 7.2 读 miss 策略配置

```json
"snapshot": {
  "read_through_timeout_ms": 10000,
  "on_timeout": "return_stale"
}
```

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `read_through_timeout_ms` | `10000` | 读穿扩展等待超时 |
| `on_timeout` | `"return_stale"` | 超时处理策略（`return_stale` / `fail`） |

### 7.3 版本一致性策略配置

```json
"snapshot": {
  "version_strategy": "auto",
  "ttl_sec": 300
}
```

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `version_strategy` | `"auto"` | 版本检测策略（`auto`/`revision`/`last_modified`/`ttl_only`） |
| `ttl_sec` | `300` | 仅 `ttl_only` 策略有效 |

## 8. 约束

1. 本文档只定义 v0.2 方向，不修改 v0.1 运行行为。
2. v0.2 文档全部中文，不产出英文同版。
3. `.act` 仅接受 JSONL 行封装，不支持原始文本直写。
4. 未通过 Phase A 冻结，不进入编码。
5. 任何协议变更必须先改文档再改实现。

## 9. 验收

1. 实施者可从 v0.2 文档直接进入开发，无需再做接口级决策。
2. v0.2 的协议、架构、CT2、实施计划文档相互引用一致。
3. 可据此直接拆分 issue backlog 并分阶段落地。
