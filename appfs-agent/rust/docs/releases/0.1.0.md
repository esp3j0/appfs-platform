# appfs-agent 0.1.0 release notes (draft)

## Summary

`appfs-agent 0.1.0` is the first release-prep checkpoint for the Rust runtime currently being adapted into AppFS's agent runtime.

At this stage, the workspace is best understood as:

- a usable local agent runtime core
- still carrying transitional internal names such as `claw`
- not yet fully integrated into the final AppFS lifecycle and control plane

## Highlights

- Rust runtime workspace established as the primary implementation surface
- interactive CLI and one-shot prompt execution
- built-in tools for shell, file operations, search, web fetch/search, todo tracking, notebooks, config, and REPL-like execution
- plugin discovery, plugin tools, and hook execution
- local agent and skill discovery
- OAuth and MCP support primitives

## Current framing

This release should be framed as:

- an early `appfs-agent` foundation release
- suitable for active development and experimentation
- not yet the final AppFS-native packaging, naming, or launch model

## Known limitations

- the current binary is still named `claw`
- source-build distribution only
- Windows build works, but Windows test readiness is still incomplete
- some advanced parity features from the historical TS snapshot remain unported
- AppFS-native runtime lifecycle integration is still ahead of this checkpoint

## Verification used for this draft

- workspace version verified from `Cargo.toml`
- current CLI surface verified from `cargo run --bin claw -- --help`
- local `cargo build --workspace` verification completed
- CI coverage verified from `.github/workflows/ci.yml`
